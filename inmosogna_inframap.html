⚠️ Nota importante: GPT-4o vs GPT-4o mini

Hay una diferencia crítica de coste entre los dos modelos de OpenAI

GPT-4o y GPT-4o mini NO son el mismo precio

GPT-4o (el modelo completo) cuesta $2.50/MTok input y $10/MTok output — más caro que Claude Haiku 3.5 ($0.25/$1.25).

GPT-4o mini cuesta $0.15/MTok input y $0.60/MTok output — el más barato del mercado y perfecto para un chatbot de cualificación de leads.

Recomendación: Usa GPT-4o mini en Sogna Core para el chatbot de WhatsApp. Para la generación de contratos de arras (más complejo, requiere precisión legal), considera GPT-4o o Claude Sonnet solo para ese endpoint específico. El coste total para 100 agencias con GPT-4o mini es de ~€5–10/mes en tokens IA.

Modelo recomendado por caso de uso

Chatbot cualificación WhatsApp: GPT-4o mini ($0.15/$0.60 MTok) — respuestas rápidas, bajo coste, suficiente para conversaciones de cualificación.

Generador contratos arras/reserva: GPT-4o ($2.50/$10 MTok) o Claude Sonnet — precisión legal importante, se usa poco (1 vez por lead cualificado).

Valorador de propiedades: GPT-4o mini con datos estructurados — análisis de comparables, no necesita razonamiento complejo.

Flujo completo de extremo a extremo

Lo que ocurre desde que un lead manda un WhatsApp hasta que el agente tiene el lead cualificado en el dashboard

1
Cliente manda WhatsApp al número de la agencia (Twilio Business Number). Twilio recibe el mensaje y lo reenvía al webhook de la InmoSogna API via HTTP POST.
2
InmoSogna API (Hono) recibe el webhook de Twilio. Valida la firma de Twilio. Identifica la agencia por el número de WhatsApp destino. Reenvía el mensaje a Sogna Core.
3
Sogna Core (FastAPI) recibe el mensaje. Recupera el historial de conversación desde Supabase. Construye el contexto para GPT-4o mini con el system prompt de cualificación inmobiliaria. Llama a la API de OpenAI.
4
GPT-4o mini responde con el siguiente mensaje de cualificación (o con el resumen del lead si ya tiene suficiente info: nombre, tipo propiedad, presupuesto, urgencia). Sogna Core guarda en Supabase y devuelve la respuesta.
5
InmoSogna API envía la respuesta al cliente via la API de Twilio WhatsApp. Si el lead está cualificado, actualiza el estado en Supabase y dispara el trigger de notificación.
6
Resend envía un email al agente: nombre del lead, teléfono, tipo de propiedad, presupuesto, urgencia y puntuación de cualificación (1–10). El agente recibe el email y entra al dashboard para ver la conversación completa.
7
InmoSogna Frontend (Next.js) muestra el lead en el dashboard, la transcripción de la conversación y opciones de acción: programar visita, generar contrato de arras o marcar como perdido.

⏱ El proceso de aprobación de Meta es el cuello de botella más largo

Meta revisa manualmente cada solicitud de cuenta de negocio para WhatsApp API. Sin el número aprobado no puedes hacer ninguna demo real. Inicia el proceso hoy aunque el código no esté listo.

Propósito en InmoSogna

Twilio actúa como middleware entre WhatsApp (Meta) y tu API. Cuando un cliente manda un WhatsApp al número de la agencia, Twilio lo recibe y lo reenvía como HTTP POST al webhook de InmoSogna API. La respuesta del chatbot se envía de vuelta al cliente via la API de Twilio.

Pasos de configuración

  1. 1
    Crear cuenta en twilio.com → verificar email y teléfono → añadir método de pago
  2. 2
    Ir a Messaging → Senders → WhatsApp Senders → clic "Request a number"
  3. 3
    Crear cuenta en Meta Business Manager (business.facebook.com) si no tienes. Necesitas: nombre del negocio (InmoSogna), sitio web (inmosogna.es), y un número de teléfono español que NO esté registrado en WhatsApp personal.
  4. 4
    Vincular la cuenta de Meta Business a Twilio siguiendo el proceso de aprobación → Twilio te guía paso a paso. Esperar 2–4 semanas la revisión de Meta.
  5. 5
    Una vez aprobado: en Twilio Console → configurar el Webhook URL del número: https://api.inmosogna.es/webhook/whatsapp
  6. 6
    Verificar que el webhook funciona enviando un mensaje de prueba al número desde tu WhatsApp personal

Alternativa si Twilio tarda: WhatsApp Cloud API directa

Meta también ofrece acceso directo a su WhatsApp Cloud API sin pasar por Twilio. El proceso de aprobación es el mismo pero sin intermediario. Coste: €0 hasta 1.000 conversaciones/mes, luego precios por conversación (~€0.05–0.08 en España). Para primeras 100 agencias puede ser más barato que Twilio.

Dos aplicaciones separadas en Fly.io

inmosogna-api: Hono + Bun — la API TypeScript que recibe el webhook de Twilio y sirve el REST API al dashboard.
sogna-core: FastAPI + Python — el motor de IA que llama a OpenAI, genera documentos y gestiona las conversaciones.

Instalación y configuración

  1. 1
    Instalar flyctl: curl -L https://fly.io/install.sh | sh → flyctl auth login
  2. 2
    En la carpeta /api: flyctl launch --name inmosogna-api --region mad (Madrid para latencia mínima en España)
  3. 3
    En la carpeta /core: flyctl launch --name sogna-core --region mad
  4. 4
    Configurar secretos: flyctl secrets set OPENAI_API_KEY=... SUPABASE_URL=... --app sogna-core
  5. 5
    Configurar dominio API: flyctl certs create api.inmosogna.es --app inmosogna-api → añadir registros DNS en Cloudflare
  6. 6
    En GitHub Actions → usar el token de Fly.io para deploy automático: flyctl deploy --remote-only

Schema de base de datos · Supabase

5 tablas con RLS para que cada agencia vea solo sus datos

-- AGENCIAS: Las cuentas que pagan la suscripción CREATE TABLE agencies ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), name TEXT NOT NULL, email TEXT UNIQUE NOT NULL, phone TEXT, city TEXT, tier TEXT DEFAULT 'basic' CHECK (tier IN ('basic', 'professional', 'elite')), stripe_customer_id TEXT, stripe_subscription_id TEXT, whatsapp_number TEXT, -- Número Twilio asignado a esta agencia active BOOLEAN DEFAULT true, created_at TIMESTAMPTZ DEFAULT NOW() ); -- LEADS: Todos los leads cualificados CREATE TABLE leads ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), agency_id UUID REFERENCES agencies(id) ON DELETE CASCADE, name TEXT, phone TEXT NOT NULL, email TEXT, operation TEXT, -- 'compra' | 'alquiler' | 'venta' | 'exclusiva' property_type TEXT, -- 'piso' | 'casa' | 'local' | 'garaje' budget_min INTEGER, budget_max INTEGER, location TEXT, bedrooms INTEGER, urgency TEXT, -- 'alta' | 'media' | 'baja' score INTEGER, -- 1–10, calculado por GPT-4o mini status TEXT DEFAULT 'new', channel TEXT DEFAULT 'whatsapp', notes TEXT, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- CONVERSATIONS: Historial de mensajes WhatsApp CREATE TABLE conversations ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), lead_id UUID REFERENCES leads(id) ON DELETE CASCADE, agency_id UUID REFERENCES agencies(id), messages JSONB DEFAULT '[]', -- [{role, content, timestamp}] status TEXT DEFAULT 'active', created_at TIMESTAMPTZ DEFAULT NOW() ); -- DOCUMENTS: Contratos generados (arras, reserva, visita) CREATE TABLE documents ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), agency_id UUID REFERENCES agencies(id), lead_id UUID REFERENCES leads(id), type TEXT CHECK (type IN ('arras', 'reserva', 'visita', 'valoracion')), file_url TEXT, -- URL de Supabase Storage created_at TIMESTAMPTZ DEFAULT NOW() ); -- VISITS: Visitas programadas CREATE TABLE visits ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), lead_id UUID REFERENCES leads(id), agency_id UUID REFERENCES agencies(id), property_address TEXT, scheduled_at TIMESTAMPTZ, confirmed BOOLEAN DEFAULT false, reminder_sent BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT NOW() ); -- RLS: Cada agencia solo ve sus propios datos ALTER TABLE leads ENABLE ROW LEVEL SECURITY; CREATE POLICY "agency_owns_leads" ON leads USING (agency_id = ( SELECT id FROM agencies WHERE email = auth.email() )); -- Repetir para: conversations, documents, visits

El stack actual llega cómodamente a 100 agencias sin cambios

Vercel escala automáticamente. Fly.io escala horizontalmente añadiendo máquinas. Supabase Pro soporta el volumen sin problemas. El único punto que necesitará atención antes de las 100 agencias es Twilio: los costes de WhatsApp suben linealmente pero siguen siendo manejables.

Cuándo escalar la infra: Cuando Supabase Free alcance 500MB de datos (migrar a Pro €25/mes) y cuando Fly.io empiece a mostrar latencia elevada en las máquinas actuales (escalar de shared-cpu-1x a performance-cpu-1x).
