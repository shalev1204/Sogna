# SOGNA - UX/UI POLICY (DESIGN LAYER)

Directrices para mantener la excelencia estética y funcional en todas las interfaces del ecosistema Sogna.

## 1. Estética Premium

- **Aesthetic Excellence**: No se permiten placeholders. Todas las interfaces deben sentirse terminadas y profesionales desde el primer despliegue.
- **Micro-animaciones**: Utiliza `animejs` o CSS transitions para mejorar el engagement sin sacrificar el rendimiento.
- **Visual Excellence**: Usa paletas de colores armónicas (HSL), modo oscuro por defecto y tipografías modernas (Inter, Roboto).

## 2. Componentización

- **Atomic Design**: Crea componentes enfocados y reutilizables.
- **Design Tokens**: Utiliza exclusivamente los tokens definidos en `intelligence/semantic/knowledge/colors.csv` y `styles.csv`.
- **Responsive Logic**: Todos los diseños deben ser Mobile-First y Touch-First.

## 3. Calidad de Activos

- **Imágenes Reales**: Genera activos reales usando `generate_image` o busca en Unsplash. Evita imágenes genéricas de baja calidad.
- **Iconografía**: Uso consistente de `Iconsax`.
