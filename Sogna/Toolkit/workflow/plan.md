# WORKFLOW: PLAN - ARCHITECTURAL BLUEPRINT

Standard procedure for designing complex features.

## Trigger
Use this workflow when a task requires more than 5 modifications or involves new architecture.

## Steps

1.  **RESEARCH**: Use `list_dir`, `grep_search`, and `view_file` to understand the impact.
2.  **DRAFT**: Outline the changes in the implementation plan artifact.
3.  **REVIEW**: Run a self-check against the relevant Rules (GEMINI, CLI, etc.).
4.  **SUBMIT**: Request feedback from the user.
5.  **FINALIZE**: Update the plan based on feedback and create the `task.md`.

## Output Elements
- Goal Description.
- User Review Required section (using github alerts).
- Proposed Changes (grouped by component).
- Verification Plan.
