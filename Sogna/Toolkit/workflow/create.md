# WORKFLOW: CREATE - FEATURE BUILDER

Standard procedure for implementing new capabilities.

## Trigger
Use this workflow when a plan has been approved and you are ready to build.

## Steps

1.  **SCAFFOLD**: Create the necessary directory structure and empty files.
2.  **FOUNDATION**: Implement core logic, data models, or design tokens.
3.  **COMPONENTS**: Build reusable UI components or utility modules.
4.  **REFINEMENT**: Apply styling, animations, and premium polish.
5.  **VERIFICATION**: Run tests and manual validation.

## Principles
- Commit frequently.
- Update `task.md` as you progress.
- Maintain documentation in real-time.
