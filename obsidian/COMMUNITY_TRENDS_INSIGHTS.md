# Tendencias de la Comunidad: Lo que el Usuario Realmente Quiere

Analizamos las peticiones más votadas en los foros de Obsidian para integrarlas de forma nativa y superior.

## 1. Bases de Datos Dinámicas (The Notion Gap)
La mayor queja es la dificultad para gestionar datos tabulares complejos en Markdown.
- **Nuestra Solución**: Integración de un motor de "Data-Views" que permita editar propiedades YAML mediante una interfaz de tabla/kanban nativa, sin depender de plugins de terceros.

## 2. Ejecución de Código y Automatización
Los usuarios quieren que sus notas "hagan cosas".
- **Nuestra Solución**: El sistema de **Skills**. Cada nota técnica puede tener un botón de ejecución que dispare una tarea en la terminal, algo que Obsidian no puede hacer por seguridad/arquitectura.

## 3. Inteligencia de Contexto Profundo
La búsqueda actual de Obsidian es rápida pero "tonta". No entiende conceptos.
- **Nuestra Solución**: Búsqueda Semántica Vectorial nativa. El sistema entenderá que una nota sobre "Seguridad en Redes" es relevante para una tarea de "Configuración de Firewall".

## 4. Colaboración y Sincronización Sin Fricción
Los usuarios odian la complejidad de Obsidian Sync o los conflictos de Git.
- **Nuestra Solución**: Un sistema de sincronización basado en agentes que resuelvan conflictos de forma semántica, no solo por línea de código.
